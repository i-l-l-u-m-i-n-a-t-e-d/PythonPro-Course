Do odświeżenia użyłem refresh tokenu otrzymanego podczas logowania.

Metoda: POST
Adres: http://127.0.0.1:8000/auth/jwt/refresh/
Body: raw, JSON

{
  "refresh": "<refresh_token>"
}

Serwer zwrócił status 200 OK oraz nowy access token

{
  "access": "<new_access_token>"
}

Następnie sprawdziłem nowy token na chronionym endpointcie

Metoda: GET
Adres: http://127.0.0.1:8000/api/protected/
Authorization: Bearer Token
Token: <new_access_token>

Odpowiedz

Status: 200 OK

{
  "username": "student1"
}

Nowy access token otrzymany z endpointu /auth/jwt/refresh/ dziala poprawnie.
